# 原生内容投放分析工作台使用说明

这是一个用于运营投放复盘的 Streamlit 网页工具。它把各渠道导出的 Excel、CSV 或 ZIP 数据统一入库，自动生成总览、渠道下钻、内容明细、数据质量、分类审核和历史报告。

工具当前重点能力：

- 上传一个周期的原始投放数据，自动标准化、入库并生成报告。
- 每个页面都可以选择分析周期，周期选择会跨页面联动。
- 总览展示核心 KPI、环比增长、分平台结果、分渠道图表和渠道栏目分析。
- 渠道下钻支持渠道 -> 二级栏目 -> 题材 Top N 分析，B站统一按 `B站全部` 处理。
- 历史报告会按批次留存，可在“生成报告”页查看、下载或基于历史原始数据重新生成。
- 支持 DeepSeek 做题材归纳和报告结论；未配置 API Key 时会自动使用本地规则兜底。
- 支持分类审核，人工确认后的分类会沉淀为历史映射。

## 1. 启动工具

推荐使用项目虚拟环境或 uv 启动。

```bash
uv run streamlit run app.py
```

如果本机没有 uv，但已经创建 `.venv`：

```bash
.venv/bin/streamlit run app.py
```

启动后在浏览器打开 Streamlit 提示的本地地址，通常是：

```text
http://localhost:8501
```

## 2. 配置 DeepSeek，可选

在项目根目录创建 `.env`：

```env
DEEPSEEK_API_KEY=你的key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
```

没有 `DEEPSEEK_API_KEY` 时，工具仍可正常生成数据报告；题材归纳和结论会使用本地规则兜底或跳过 DeepSeek 结论。

## 3. 准备原始数据

推荐每个周期准备一个文件夹，里面放各渠道导出的原始文件：

```text
data/raw/20260518-20260524/
  B站.xlsx
  小红书商业化.xlsx
  抖音商业化.xlsx
  抖音市场部.xlsx
```

也可以直接在网页上传：

- 单个或多个 CSV / XLS / XLSX 文件
- 一个 ZIP 数据包
- 一个包含多个文件的文件夹

文件名会优先作为渠道名。B站、小红书、抖音有专门的字段读取规则，其他渠道会按通用字段候选读取。所有原始 Excel 都会先清洗为 `cleaned.xlsx`，再进入统计分析；每个 workbook 的 sheet 会记录在 `导入日志` 或 `忽略sheet` 中，避免静默跳过。

## 4. 生成报告

进入左侧导航的“生成报告”。

1. 在“上传原始数据”中选择目录，或把多个 CSV / Excel / ZIP 拖入上传区。
2. 确认原始数据已上传。
3. 选择周期开始和周期结束。
4. 点击 `生成并存档`。

生成成功后，工具会写入这些位置：

- `data/raw/{YYYYMMDD-YYYYMMDD}/`：页面上传后的周期原始目录。
- `archive/{period_start}_{batch_id}/raw/`：本次生成使用的归档原始数据。
- `outputs/{batch_id}/report.html`：HTML 报告。
- `outputs/{batch_id}/analysis.xlsx`：分析明细。
- `outputs/{batch_id}/canonical.csv`：标准化明细。
- `outputs/{batch_id}/total_summary.xlsx`：总结果表。
- `data/workflow.sqlite3`：历史批次、标准化明细、分类、质量检查和对比数据。

周期原始目录中还会生成：

- `cleaned.xlsx`：统一清洗后的标准明细。
- `period_manifest.json`：本周期清洗清单。

`cleaned.xlsx` 包含 `清洗后明细`、`导入日志`、`忽略sheet`、`重复文件`、`重复内容`、`冲突项` 等工作表。分析只从 `清洗后明细` 读取，不再绕过清洗日志直接读取原始 Excel。

## 5. 查看或更新历史报告

“生成报告”页下方有“历史报告”区域。

可执行三类操作：

- `查看历史报告`：在页面内预览历史批次的 `report.html`。
- `下载历史报告`：下载 HTML 报告、Excel 明细、标准 CSV、总结果表。
- `更新报告`：基于该批次归档的 `archive/.../raw/` 原始数据重新生成一个新批次。

更新不会覆盖旧报告。旧批次继续保留，新结果会生成新的 `batch_id`、新的 `outputs/{batch_id}/` 和新的数据库记录。

## 6. 页面功能说明

### 总览

用于快速看当前周期整体表现。

- `总体核心指标`：总消耗、激活数、激活成本、付费数、付费成本、付费率。
- `环比增长`：按周期日期查找上一个更早成功周期，展示消耗、激活、付费和成本变化；不按入库时间决定上一期。
- `分平台核心结果`：按渠道展示核心指标。
- `直接建议`：基于题材、渠道和成本的文本建议。
- `分渠道表现`：可切换消耗、激活数、付费数、激活成本、付费成本、付费率。
- `渠道栏目分析`：选择渠道后查看该渠道下二级栏目的图表。

页面顶部的周期选择使用北京时间显示，格式为：

```text
YYYY年MM月DD日  HH:mm:ss
```

### 渠道下钻分析

用于按“渠道 -> 栏目 -> 题材”定位问题或机会。

流程：

1. 选择周期。
2. 选择渠道。
3. 选择排序指标。
4. 设置 Top N，默认 15，可调 5-50。
5. 查看栏目分布和题材 Top N。

B站暂不做栏目层级拆分，统一使用 `B站全部`，直接进入题材分析。

### 内容类型统计

查看指定周期内各内容类型的素材数、唯一内容数、消耗、激活、付费和成本表现。

### 历史趋势

查看多个历史周期的趋势变化。支持快捷日期范围：

- 全部
- 一周
- 两周
- 一个月

### 内容明细

查看标准化后的内容级明细。适合按标题、内容 ID、素材 ID、账号、作者或题材搜索具体记录。

### 数据质量

查看导入过程中的异常，包括缺失字段、账号缺失、分类缺失、冲突保留、需要复核的数据等。

### 维护台账

查看本地维护表，来源为：

```text
config/reference_tables.xlsx
```

如果文件不存在，工具会自动创建初始版本。常用表包括：

- 内容类型分级表
- 账号内容类型对照表
- 账号映射表
- 字段映射表
- 处理规则

### 分类审核

处理需要人工确认的内容分类。确认保存后，会写入分类映射，后续批次会复用。

### 历史批次

查看所有历史批次的周期、生成时间、输出目录和对比说明。

## 7. 字段口径

核心字段说明：

- `渠道`：每个原始文件对应一个渠道，例如 `B站`、`小红书商业化`、`抖音商业化`。
- `账号ID`：保存 B站 MID/UID、小红书号、抖音号等原始账号标识。
- `实际账号`：来自原始表、默认维护或账号映射表。
- `账号映射来源`：标记为 `原始账号字段`、`默认维护`、`账号映射表` 或 `未匹配`。
- `二级栏目`：当前主要分类层级。B站统一为 `B站全部`。
- `三级题材`：更细的题材层级，可由标题、标签、内容 ID、本地规则或 DeepSeek 归纳得到。
- `人工内容类别`：来自原始表或人工审核。
- `AI生成内容类别`：人工类别缺失时由规则或 DeepSeek 生成。
- `最终内容类别`：汇总分析使用的类别，优先人工，其次 AI 或规则。
- `内容类别来源`：标记分类来源和置信度。

账号统计范围：

- 小红书只统计这些原始账号：`同花顺投资`、`同顺股民社区`、`同花顺理财`、`同顺财经`、`问财`、`喵懂投资`。
- 抖音和 B站当前不做账号过滤，原始表里能识别到的行会进入对应渠道统计；短账号名和空账号不会因为账号白名单被排除。
- 抖音账号白名单和 B站 `Up主mid=1622777305` 映射规则保留在 `config/account_filters.yml`，但当前 `filter_enabled: false`，只作为后续启用时的配置基础。
- 小红书短账号名可以继续使用，例如 `股民社区`、`投资号`、`理财`、`财经号`、`问财`，系统会按 `config/account_filters.yml` 归一到上面的原始账号口径。
- 小红书原始表没有账号名的行不进入小红书统计，会进入账号过滤明细。
- 被过滤的账号行不会参与消耗、激活、付费和成本汇总，可在分析工作簿的数据质量或账号过滤明细中查看原因。

核心指标说明：

- `消耗`：投放花费。
- `激活数`：激活用户数。
- `激活成本`：消耗 / 激活数。
- `付费数`：首次付费次数或应用内付费次数，按渠道字段映射统一。
- `付费成本`：消耗 / 付费数。
- `付费率`：付费数 / 激活数。

## 8. 自动同步 raw 目录

网页运行时会定时检查 `data/raw/` 下的周期目录。如果发现合法周期目录里的原始文件新增或变化，会自动生成新批次。

合法目录名示例：

```text
data/raw/20260518-20260524/
```

如果不想自动生成，请不要把未确认的数据放进 `data/raw/` 的合法周期目录。

## 9. 命令行生成

除了网页，也可以用命令行生成。

生成并入库：

```bash
uv run python main.py \
  --input data/raw/20260518-20260524 \
  --period-start 2026-05-18 \
  --period-end 2026-05-24 \
  --output outputs \
  --archive-root archive \
  --db data/workflow.sqlite3
```

只生成文件、不写数据库：

```bash
uv run python main.py \
  --input data/raw/20260518-20260524 \
  --period-start 2026-05-18 \
  --period-end 2026-05-24 \
  --output outputs/manual \
  --legacy-output-only
```

## 11. 常见问题

### 没有 DeepSeek Key 能不能用？

可以。工具会继续生成数据报告，DeepSeek 相关能力会降级为本地规则。

### 历史报告会被覆盖吗？

不会。每次生成都会产生新的批次目录和数据库记录。历史报告可以在“生成报告 -> 历史报告”中查看、下载或基于旧原始数据重新生成新批次。

### 为什么有些分渠道环比为空？

如果上一周期没有同名渠道，或上一周期该指标为 0，就无法计算环比。总计环比仍会在有历史数据时展示。

### B站为什么没有栏目选择？

B站当前统一按 `B站全部` 处理，不拆二级栏目。渠道下钻时会直接进入题材分析。

### 上传后标题缺失但数据很高怎么办？

渠道下钻和异常检测会标记标题缺失但消耗、激活或付费较高的记录，便于后续回查原始数据。
