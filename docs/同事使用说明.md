# 同事使用说明

这份说明面向日常使用，不需要懂代码。系统用于把多个渠道 Excel 清洗成一份统一结果，并在网页里查看图表和生成 AI 复盘。

## 1. 打开系统

如果系统已经启动，直接在浏览器打开负责人给你的局域网地址：

```text
http://本机局域网IP:端口
```

如果需要在这台电脑上启动，打开项目文件夹后双击：

- Mac 电脑：`启动局域网.command`
- Windows 电脑：`启动局域网.cmd`

启动窗口会自动检查环境。缺少 `uv`、Python 或依赖时，会从大陆镜像自动下载；窗口里会持续显示中文进度，首次启动时间较长时不要关闭。

默认使用 `8501` 端口。如果 `8501` 被占用，系统会自动改用后续可用端口，例如 `8502`。请以启动窗口最后显示的地址为准：

```text
本机访问：http://127.0.0.1:8502
局域网访问：http://本机局域网IP:8502
```

使用期间不要关闭启动窗口。需要关闭系统时，回到启动窗口按 `Ctrl+C`，看到“服务已关闭，可以关闭窗口”后再关闭。

命令行备用方式：

```bash
uv run streamlit run app.py
```

如果上面命令不可用，使用：

```bash
.venv/bin/streamlit run app.py
```

## 2. 上传 Excel 规范

进入左侧“生成页面数据”。

可以上传：

- 一个或多个 Excel 文件。
- CSV 文件。
- ZIP 压缩包。
- 包含多个文件的文件夹。

文件名尽量写清楚周期和渠道，例如：

- `2026年5月-小红书商业化.xlsx`
- `2026年5月-小红书市场部.xlsx`
- `2026年5月-抖音商业化.xlsx`
- `2026年5月-抖音市场部.xlsx`
- `2026年5月-B站.xlsx`
- `2026年5月-B站商业化.xlsx`
- `2026年5月-微信市场部.xlsx`
- `2026年5月-微信商业化.xlsx`

## 3. 同周期同渠道覆盖规则

如果你上传的是同一周期、同一渠道的新文件，系统会提示已有文件。

- 确认覆盖：旧原始文件会被新文件替换，然后系统重新清洗并刷新页面数据。
- 不覆盖：旧文件保留，新文件不会写入。

覆盖只影响当前周期当前渠道，不影响其他周期和其他渠道。

## 4. 生成清洗结果

上传后选择周期，点击“生成本周期数据”。

生成完成后，系统会在后台生成统一清洗结果：

```text
processed/周期/batch_id/cleaned_channels.xlsx
```

这份 Excel 是主要交付物，一个渠道一个 sheet。系统 sheet 包括：

- `导入日志`：哪些文件和 sheet 被读取。
- `重复内容`：自动去重记录。
- `冲突项`：同 ID 标题冲突、同标题多链接等问题。
- `补齐来源`：字段是从原始表、台账、公开信息或 AI 候选补齐的。
- `审核记录`：人工审核保存后的记录。

每个渠道 sheet 会先展示统一字段，再保留这个渠道实际出现的未映射原始字段。原表头是什么，清洗结果里就用什么列名，例如“7日付费成本”仍显示为“7日付费成本”。

## 5. 渠道配置方法

进入左侧“维护台账”，查看“渠道配置说明”。

默认渠道包括：

- 小红书商业化
- 小红书市场部
- 抖音市场部
- 抖音商业化
- B站市场部
- B站商业化
- 微信市场部
- 微信商业化

如果以后新增平台或渠道，优先让负责人在 `config/channel_profiles.yml` 里新增一条配置。需要准备的信息：

- 渠道名。
- 平台名。
- 文件名关键词。
- 这个渠道常见字段别名。
- 是否启用账号过滤。

## 6. 人工审核步骤

进入左侧“内容审核”。

系统只会把最高优先级的少量重点内容放进审核队列，每个周期少于 50 条。

优先进入审核的内容通常是：

- 高消耗且标题、链接、内容类型等有冲突。
- 高消耗且内容链接、内容 ID、标题、内容类型补齐失败。
- 各渠道高消耗内容中仍有关键字段缺失的条目。
- 其他高消耗重点内容。

完整的冲突和补齐记录仍保留在清洗结果 workbook 的系统 sheet 中。

你需要做的事：

1. 查看当前条目的标题、链接、消耗和审核原因。
2. 补齐或确认内容链接。
3. 确认内容类型。
4. 点击“保存并下一条”。

保存后系统会自动同步最终数据、图表和手动 AI 复盘输入。

## 7. 查看图表

生成完成后，可进入：

- “总览”：看总消耗、曝光、激活、付费、成本和环比。
- “渠道下钻分析”：看单个渠道的内容表现。
- “历史趋势”：看多个周期的趋势变化。

各页面顶部都可以选择周期。

## 8. 手动 AI 复盘

进入“总览”页，确认数据无误后点击手动 AI 复盘按钮。

注意：

- 上传和清洗不会自动生成 AI 报告。
- 审核保存后再生成 AI 复盘，报告会使用最新结果。
- 如果没有配置模型 key，页面会提示无法生成 AI 报告，但清洗和图表不受影响。

## 9. 常见问题

### 上传后找不到渠道

检查文件名是否包含渠道关键词，例如“小红书商业化”“抖音市场部”“B站商业化”。如果是新渠道，系统会先按文件名生成一个新渠道，平台显示为“其他”；后续再让负责人把它加入渠道配置。

### 同一个周期数据传错了

重新上传正确文件，选择同一周期同一渠道，确认覆盖，然后重新生成数据。

### 清洗结果里有字段为空

先看 `cleaned_channels.xlsx` 的 `补齐来源` 和 `冲突项`。如果系统无法自动补齐，会进入内容审核页。

### 图表没有刷新

确认是否点击了“生成本周期数据”或在内容审核页保存后等待同步完成。必要时切换周期再切回来。

### AI 复盘内容不是最新

请先保存审核结果，再回到“总览”手动重新生成 AI 复盘。

### 局域网打不开

确认启动电脑和访问电脑在同一个网络，并使用启动窗口最后显示的局域网地址访问。端口默认是 `8501`，但如果 `8501` 被占用，系统会自动切换到 `8502`、`8503` 等后续端口。
