# 领导口头交流汇报稿

## 汇报定位

本次汇报面向项目进展沟通，重点说明三件事：

- 已经完成了什么：系统已经把投流数据接入、清洗、去重、匹配飞书自有内容、图表和手动复盘串成基础闭环。
- 下一步怎么升级：从复盘工具升级为内容选材和投放策略辅助决策工具。
- 当前卡点在哪里：数据端字段、账号、分类和渠道粒度还不够稳定，会直接影响推荐结论可信度。

核心主线：

> 项目已经完成从“投流数据复盘”到“下一周期内容选材建议”的基础闭环，但要真正稳定指导投放，还需要先补齐数据口径和分类治理。

## 口头开场

这个项目不是单纯做一张投放报表，而是把各渠道投流数据统一处理后，沉淀成可复盘、可比较、可追踪的内容选材依据，最终服务下一周期选题和预算分配。

目前系统已经能把原始投放数据统一清洗入库，并生成总览、上传清洗、高价值内容复盘、本地总表、周期数据和历史趋势六个主入口，也能基于高价值内容的渠道和内容类型表现沉淀下一轮内容方向。

但我也想强调一个前提：系统能力已经搭起来了，真正决定后续建议质量的，是数据源字段是否稳定、分类标准是否统一、账号和内容 ID 是否能长期追踪。

## 已完成内容

第一，数据入口已经打通。

现在支持 Excel、CSV、ZIP、文件夹上传，也支持命令行生成。运营同学可以直接把一个周期的原始投流数据放进系统，系统自动保存原始文件、清洗入库并生成 `content_recap_core.xlsx`。

第二，标准化链路已经完成。

不同渠道的字段口径不一样，比如消耗、曝光、点击、激活、付费、账号、内容 ID、素材 ID、内容类型等，在系统里已经被统一成一套分析字段。这样后面做渠道对比、内容类型分析和复盘建议时，不再依赖人工拼表。

第三，工作台页面已经成型。

当前工作台已经收敛为总览、上传清洗、高价值内容复盘、本地总表、周期数据和历史趋势。也就是说，既能看整体表现，也能围绕高价值素材查看类型占比、素材缓存和复盘结论。

第四，推荐雏形已经具备。

系统现在可以按渠道和内容类型查看消耗、激活、付费、激活成本、付费成本和付费率，并结合热度评分、拉新评分输出下一周期内容建议。这个能力后续可以升级成更明确的放量、观察、暂停、补数建议。

第五，数据治理能力已经接入流程。

缺失字段、异常指标、账号缺失、冲突和补齐失败不会被系统静默吞掉，而是会进入核心复盘 workbook 的清洗后素材表和不可分析汇总。飞书台账或维护映射更新后，后续批次可以复用。

第六，工程验证已经覆盖核心流程。

当前测试覆盖上传、入库、历史批次、环比、下钻、清洗、补齐和审核等核心流程。

## 当前数据端问题

第一，渠道导出字段不统一。

不同平台对标题、账号、内容分类、点击、付费等字段命名和缺失情况不同。系统能做适配，但如果源头字段长期变化，后续自动归因会受到影响。

第二，账号维度不稳定。

B站常见情况是只有 MID，没有账号名；抖音导出样例里也可能缺账号名。系统现在不会把平台名称或数字 ID 猜成账号名，只会使用原始导出字段或本地总表里的映射信息，这样更稳，但也意味着账号映射需要补齐。

第三，自有内容台账和分类口径仍需治理。

系统只把已匹配飞书自有内容的素材纳入内容复盘。未匹配素材不再交给 AI 强行分类，而是统计不可分析占比和原因；后续治理重点是补齐飞书台账中的作品链接、作品 ID/BV号和内容类型。

第四，部分渠道分析粒度有限。

例如 B站当前只默认内容形式为视频，内容类型缺失时留空，适合先做渠道级复盘；内容类型拆分能力仍弱于抖音、小红书这类字段更丰富的平台。

第五，推荐结论依赖数据完整度。

如果标题、内容类型、账号或稳定 ID 缺失，系统仍然可以生成报告，但下一周期内容建议的可信度要打折。换句话说，系统可以自动分析，但不能替代数据源治理。

## 未来升级方向

第一，从“看数据”升级为“给动作”。

现在系统能告诉我们哪些渠道和内容类型表现好。下一步要把结论变成动作建议，例如放大、观察、暂停、补数四类，并给出推荐理由和置信度。

第二，从单周期复盘升级为闭环学习。

后续可以记录本期推荐了哪些内容方向，下周期回看这些方向是否真的跑赢。如果跑赢，就提高相应评分权重；如果没有跑赢，就反向修正评分逻辑。

第三，强化数据规范。

建议固定各渠道导出模板，明确必填字段，包括内容 ID、素材 ID、标题、账号、分类、曝光、点击、激活和付费。这个工作会直接决定系统能否从“能用”变成“稳定可用”。

第四，强化分类资产。

建立统一内容类型库，减少临时分类、重复分类和 AI 自由发挥。后续所有推荐都尽量基于受控内容类型输出，避免同一类内容被分散到多个近似名称里。

第五，强化自动化。

后续可以增加定时同步、异常提醒、自动生成汇报摘要，进一步扩展成周报或经营看板。这样领导看到的不只是一次性报告，而是持续更新的投放决策面板。

## 结尾口径

当前项目已经把分析链路搭起来了，下一步重点不是继续堆页面，而是把数据源和分类标准做稳。

只要数据端字段稳定，系统就能从复盘工具升级成内容选材和投放策略的辅助决策工具，帮助我们在下一周期更快判断应该放大什么、减少什么、补充什么数据。

## 可能追问与回答

### 领导问：现在能不能直接指导下一周期选材？

可以作为参考，但建议分两层看。

第一层，渠道和内容类型的表现趋势已经能提供方向判断。第二层，如果要作为预算和选题的强决策依据，还需要确保正式周期数据完整，尤其是账号、标题、内容类型和内容 ID 稳定。

### 领导问：为什么不让 AI 给所有素材分类？

内容复盘的前提是确认素材对应我们自己创作的内容。AI 可以辅助已匹配内容的复盘表达，但不能替代作品身份匹配；未匹配飞书自有内容的素材只统计不可分析占比和原因。

### 领导问：这个项目后续最重要的投入是什么？

最重要的是数据标准化和分类资产建设。页面和图表已经具备基础能力，后续真正提高决策价值的是让各渠道数据字段稳定、内容分类统一、账号和内容可以跨周期追踪。

### 领导问：怎么衡量这个项目有没有价值？

可以用三个指标衡量：

- 复盘效率：从人工整理数据到自动生成报告，节省多少时间。
- 推荐命中率：系统推荐的内容方向在下一周期是否跑赢平均成本或付费效率。
- 数据质量改善：标题、账号、内容类型、内容 ID 等关键字段缺失率是否下降。
